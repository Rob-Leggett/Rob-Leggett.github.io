Folks have been asking me if I knew of any good examples of a modern full-stack web application that utilises the following technologies: **Angular 19**, **Bootstrap 5.3**, **Spring Boot 3.3**, **Spring Security 6**, **HSQLDB**, and **JWT stateless authentication**, and after searching around I decided that it would be best to create an example project that would demonstrate the integration of all these technologies as well as describe some of the specifics.

This project has been fully modernised from its original AngularJS and Spring 4 roots to reflect current industry standards, running on **Java 21** and **TypeScript 5.7** with a **Maven multi-module** structure.

## Angular 19

### Why choose this technology

For the example I wanted to create a client-rich web application that uses a modern, widely adopted front-end framework.

**Angular 19** with **standalone components** provides a powerful, type-safe development experience with built-in routing, dependency injection, and reactive forms — all without the overhead of NgModules.

Angular continues to be one of the most sought-after frameworks on the market, and the move to standalone components represents the modern way to build Angular applications.

### Technology focus areas

*   **Angular 19** with standalone components
    *   Component-based architecture
    *   Angular Router with lazy loading
    *   Template-driven and reactive forms
    *   HttpClient with interceptors for JWT token injection
    *   RxJS observables and reactive patterns
    *   Services and dependency injection
    *   HTTP response interceptors to catch 401 responses and redirect to login
    *   Event-driven communication between components
*   **TypeScript 5.7** for type safety
*   **Angular CLI** for build tooling and development server
*   **JWT token management** on the client side

---

## Bootstrap 5.3

### Why choose this technology

For the example I wanted to create a web application that looks professional and is quick to style.

**Bootstrap 5.3** is a responsive web library allowing for ideal UI display across different devices and resolutions. The move from Bootstrap 3 to 5 drops the jQuery dependency entirely, aligning with Angular's component-driven approach.

### Technology focus areas

*   **Bootstrap 5.3**
    *   NavBar
    *   Header
    *   Dropdown
    *   Cards (replacing the old Panel component)
*   Responsive Web Layouts
*   No jQuery dependency

---

## Spring Boot 3.3

### Why choose this technology

For the example I wanted to create the functionality scenarios to best emulate industry best practices and for the reader to be able to walk away understanding how they can utilise modern server-side technologies that are currently strong in the market.

I decided to use **Spring Boot 3.3** to create a **RESTful API** in which the client could GET, POST, PUT, DELETE data as required. Spring Boot provides auto-configuration, embedded Tomcat, and a streamlined development experience compared to the traditional XML-heavy Spring configuration of the past.

### Technology focus areas

*   **Spring Boot 3.3** with auto-configuration and embedded Tomcat
*   **Java 21** as the runtime, taking advantage of modern language features
*   Annotations are used throughout to configure controllers, services, transactions, and components
*   RESTful entry points are defined through controllers and produce JSON responses
*   CRUD operations
*   **JUnit 5** test support

---

## Spring Security 6

### Why choose this technology

For the example I wanted to have the RESTful API endpoints protected as sensitive information can be accessed and sensitive operations performed.

**Spring Security 6** with **JWT stateless authentication** provides a modern, token-based approach to securing your API. Instead of session-based authentication, the server issues a JWT on login and validates it on each subsequent request.

### Technology focus areas

*   **Stateless JWT authentication** — no server-side session required
*   **SecurityFilterChain** bean-based configuration (replacing the legacy WebSecurityConfigurerAdapter)
*   Method-level security with @PreAuthorize and @PostAuthorize annotations
*   Token-based authentication flow (login → receive JWT → include in Authorization header)
*   An entry point that returns HTTP 401 for unauthenticated requests
*   **BCrypt** password encoding
*   JUnit 5 test support

---

## HSQLDB

### Why choose this technology

For the example I wanted to demonstrate a complete end-to-end solution where transaction operations could be executed that would show industry-based standards.

**HSQLDB** provides a lightweight, in-memory relational database that requires no external setup, making it ideal for a starter project that anyone can clone and run immediately.

### Technology focus areas

*   **Spring Data JPA** with HSQLDB in-memory database
*   Auto-configuration via Spring Boot — no persistence.xml required
*   Entity Manager and simple transactions managed by Spring
*   Zero configuration for database setup
*   JUnit 5 test support

---

## Maven Multi-Module Structure

The project is organised as a **Maven multi-module** build with two modules:

*   **api** — the Spring Boot 3.3 back-end with REST endpoints, Spring Security 6, and JPA persistence
*   **client** — the Angular 19 front-end, built via Angular CLI and served alongside the API

This structure provides a clean separation of concerns while allowing a single build command to produce the full application.
